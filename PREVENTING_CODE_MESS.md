# Preventing Code Mess (skrót lokalny)

## Jak ograniczać narastanie długu

- Jedna zmiana = jeden cel.
- Nie mieszaj refaktoru i nowej funkcji.
- Nie dotykaj niezależnych subsystemów „przy okazji”.
- Utrzymuj spójność między dokumentacją i stanem kodu.

## Praktyka

- Przed zmianą przeczytaj lokalne typy, helpery i testy.
- Po zmianie uruchom testy celowane, a potem pełny zestaw.
- Aktualizuj checklistę i dokumentację statusową.
