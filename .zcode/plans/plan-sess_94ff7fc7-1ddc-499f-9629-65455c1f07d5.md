# Splash screen na wszystkich platformach DINBoard

## Wybrane (z Twoich odpowiedzi)
- **Platformy:** Web/PWA + Native Capacitor (iOS/Android) + iOS Safari PWA
- **Kiedy:** wszędzie przy starcie aplikacji
- **Wygląd:** dokładnie jak na zrzucie — granat `#0e0f11`, żółte logo DIN na środku, „DINBoard" („Board" w amber), subtitle „APLIKACJA DLA ELEKTRYKÓW", pasek ładowania na dole

---

## Diagnoza (stan obecny — z badania kodu)

| Platforma | Stan dziś |
|---|---|
| **Web/PWA** | ❌ pusty `<div id="root">` aż do hydratacji React. Na `/app` goły tekst „Ładowanie edytora..." w kolorze `#888`. |
| **Capacitor native** | ⚠️ splash PNG istnieje (11 Android + 3 iOS) ale to **stockowy placeholder Capacitora** (niezbrandbizowany). `capacitor.config.ts` ma **zero** konfiguracji `SplashScreen`. Plugin `@capacitor/splash-screen` zainstalowany ale nieużywany w JS. |
| **iOS Safari PWA** | ❌ brak `apple-touch-startup-image` w `index.html` — po dodaniu do ekranu głównego i uruchomieniu widać białe migające tło przed załadowaniem React. |

## Architektura

Trzy osobne implementacje (każda żyje w innym miejscu kodu, nie da się tego scalić):

### A. Web/PWA — inline splash w `index.html` (główna realizacja)

**Dlaczego w index.html, nie w React:** React ładuje się ~200-500ms po pierwszym paint. Splash w React by się pojawił ZA PÓŹNO — użytkownik widziałby puste tło. Inline HTML+CSS w `<body>` renderuje się w pierwszej klatce, jeszcze zanim JS się pobierze.

**Realizacja:**
- Nowy blok `<div id="app-splash">` w `<body>` `index.html`, PRZED `<div id="root">`
- Cały CSS splash inline w `<style>` w `<head>` (żeby nie czekać na zewnętrzny CSS)
- Z-index wyżej niż `#root` (splash przykrywa root aż do ukrycia)
- **Logo:** `<img src="/dinboard.svg">` (czysty wektor, świetna jakość na każdym ekranie)
- **Tekst:** „DINBoard" z `Board` w kolorze amber `#f59e0b` (split-color wordmark jak w `LandingHeader`)
- **Subtitle:** „APLIKACJA DLA ELEKTRYKÓW" letter-spacing, muted
- **Pasek ładowania:** animowany CSS gradient `@keyframes` (3 paski lub 1 skanujący, jak na zrzucie)
- **Ukrywanie:** skrypt w `main.tsx` (po hydratacji React + `useEffect`) usuwa splash z `fade-out` animacją. CSS transition opacity 1→0 przez 300ms, potem `display: none`.

**Pliki do zmiany:**
- `index.html` — dodać `<style>` w head + `<div id="app-splash">` w body + skrypt usuwający
- `src/main.tsx` — po `ViteReactSSG` callback lub w `AppLayout.useEffect` ukryć splash
- `src/App.tsx` — `AppLayout` dodaje klasę `app-ready` do `<html>`, CSS ukrywa splash

### B. Capacitor native — konfiguracja + nowe PNG

**Realizacja:**
- Skrypt Node/Python rasteryzujący splash do PNG w odpowiednich rozmiarach:
  - Android: `drawable-*/splash.png` (5 density bucketów × 2 orientacje = 10 plików)
  - iOS: `Splash.imageset/splash-2732x2732.png` (2732×2732 uniwersalny)
- Tło `#0e0f11` + wycentrowane logo DIN (skrypt PIL lub generacja przez `<canvas>` w skrypcie build)
- `capacitor.config.ts` — dodać blok `plugins.SplashScreen`:
  ```ts
  plugins: {
    SplashScreen: {
      launchShowDuration: 2000,
      backgroundColor: "#0e0f11",
      showSpinner: false,
      androidSplashResourceName: "splash",
      androidScaleType: "CENTER_CROP",
      splashFullScreen: true,
      splashImmersive: false
    }
  }
  ```
- `src/main.tsx` lub dedykowany `src/lib/native/splash.ts` — `SplashScreen.hide()` gdy React gotowy (zapobiega „zawieszeniu" splash po załadowaniu webview)
- **UWAGA na high-risk:** zmiana `capacitor.config.ts` jest w strefie Project I/O wg AGENTS.md. Dodaję tylko `plugins.SplashScreen` — nie ruszam `appId`, `webDir`, `appName`.

**Pliki:**
- `capacitor.config.ts` — dodać blok `plugins.SplashScreen`
- `android/app/src/main/res/drawable*/splash.png` — wygenerować nowe (10 plików)
- `ios/App/App/Assets.xcassets/Splash.imageset/splash-2732x2732*.png` — wygenerować nowe (3 pliki)
- `src/lib/native/splash.ts` (nowy) — wrapper `hideNativeSplash()` z try/catch (web build nie ma pluginu)
- `src/main.tsx` — wołać `hideNativeSplash()` po mount
- `scripts/generate-native-splash.mjs` (nowy) — skrypt do regeneracji PNG

### C. iOS Safari PWA — `apple-touch-startup-image`

**Realizacja:**
- Generacja PNG startup image w kluczowych rozmiarach iPhone'a:
  - 320×448 (iPhone SE / non-retina)
  - 640×960 (iPhone 4/4s)
  - 750×1294 (iPhone 6/7/8 portrait)
  - 1242×2148 (iPhone 6/7/8 Plus portrait)
  - 1125×2436 (iPhone X-XS portrait)
  - 828×1792 (iPhone XR/XS Max)
  - 2048×2732 (iPad Pro 12.9")
- W `index.html` dodać linki `<link rel="apple-touch-startup-image" media="(device-width: ...)" href="...">` (6-8 wpisów)
- Tło `#0e0f11` + wycentrowane logo (PIL skrypt)

**Pliki:**
- `public/assets/splash/ios-startup-*.png` — wygenerowane PNG (6-8 plików)
- `index.html` — `<link rel="apple-touch-startup-image">` z media queries
- `scripts/generate-ios-startup.mjs` — skrypt regeneracji
