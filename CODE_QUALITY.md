# Code Quality (skrót lokalny)

## Zasady

- Preferuj małe, bezpieczne zmiany.
- Unikaj refaktorów szerokiego zakresu przy małej poprawce.
- Zostaw kod czytelniejszy niż zastałeś, bez zmian semantyki.
- Dodawaj testy, gdy zmieniasz logikę biznesową.

## Obszary krytyczne

- walidacja elektryczna
- bilans faz
- relacje RCD -> MCB/RCBO
- zapis/odczyt projektu
- eksport PDF

Szczegółowe reguły: `AGENTS.md`.
